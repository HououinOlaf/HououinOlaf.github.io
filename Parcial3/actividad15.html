# 01_RESUMEN_EJECUTIVO_Y_ALCANCE

> Este reporte documenta la actividad "Creando una campaña de phishing: SET (Social Engineer Toolkit)", desarrollada en un contexto escolar para la asignatura de Seguridad Informática[cite: 24].

> La práctica consistió en diseñar y enviar correos simulados de phishing mediante SET, con el propósito de observar cómo un atacante puede explotar factores humanos como autoridad, urgencia y miedo a consecuencias académicas[cite: 25]. La campaña simulada utilizó mensajes relacionados con la seguridad de la cuenta, límite de faltas e irregularidades de reinscripción[cite: 26].

> En cada caso se analizaron los elementos que aumentan la credibilidad del engaño, los indicadores técnicos visibles y las medidas de prevención aplicables a estudiantes e instituciones[cite: 27].

// LÍMITES Y CONSIDERACIONES ÉTICAS

La actividad se considera válida únicamente bajo un contexto académico controlado[cite: 33]. Ejecutarla sin autorización puede afectar cuentas, datos personales, infraestructura institucional y confianza de los usuarios[cite: 35]. Los límites de seguridad establecidos fueron:

  • > No almacenar contraseñas reales ni información de sesión de usuarios[cite: 46].
  • > No utilizar dominios, logos o cuentas institucionales reales sin autorización expresa[cite: 47].
  • > No enviar campañas masivas fuera del grupo o alcance aprobado[cite: 48].
  • > No publicar evidencias con datos personales en espacios abiertos[cite: 49].
  • > Orientar el reporte a defensa, concientización y mejora de controles[cite: 50].

# 02_MARCO_TEÓRICO_Y_VECTORES_DE_ATAQUE

// PHISHING E INGENIERÍA SOCIAL

El phishing es una técnica de ingeniería social en la que el atacante intenta engañar a una persona para que realice una acción perjudicial: ingresar credenciales, abrir enlaces, descargar archivos, autorizar accesos o revelar información[cite: 62]. La ingeniería social no depende únicamente de vulnerabilidades técnicas, su fuerza está en manipular decisiones humanas[cite: 65].

En las evidencias de esta práctica se observan tres disparadores psicológicos relevantes[cite: 66]:

  • > Autoridad: el mensaje aparenta provenir de servicios escolares o seguridad institucional[cite: 67].
  • > Urgencia: se establece una ventana de tiempo reducida para actuar[cite: 71].
  • > Miedo a pérdida: se amenaza con suspensión de cuenta, baja temporal o pérdida de derecho a examen[cite: 72].

// THE SOCIAL-ENGINEER TOOLKIT (SET)

SET es una herramienta utilizada en laboratorios de seguridad para simular escenarios de ingeniería social[cite: 74]. Permite construir vectores de ataque controlados como correos, páginas clonadas, payloads o enlaces[cite: 75].

En esta práctica se utilizó el módulo de correo electrónico (Mass Mailer Attack), una de las modalidades más comunes porque permite distribuir mensajes con apariencia institucional, incluir enlaces externos y presionar al usuario mediante asuntos urgentes[cite: 78]. El riesgo no radica solo en la herramienta técnica, sino en la combinación entre un pretexto creíble, la identidad aparente del remitente y la falta de validación del destinatario[cite: 79].

# 03_DISEÑO_DE_LA_CAMPAÑA_SIMULADA

> La campaña se construyó con tres variantes de mensaje. Todas mantienen una temática escolar porque ese contexto resulta creíble para estudiantes universitarios y aprovecha procesos que suelen generar presión[cite: 81].

Variante Pretexto de Ingeniería Social Riesgo Buscado (Objetivo del Atacante) Nivel de Riesgo Estimado
Seguridad de cuenta Aviso de actividad inusual y supuesta vinculación obligatoria de seguridad[cite: 82]. Inducir al usuario a validar identidad mediante un enlace externo[cite: 82]. Alto: Combina autoridad tecnológica, amenaza de inhabilitación y solicitud de validación[cite: 259].
Límite de faltas Alerta de inasistencias superiores al porcentaje permitido[cite: 82]. Generar urgencia por miedo a perder derecho a examen ordinario[cite: 82]. Alto: Ataca un miedo académico directo. El plazo corto incrementa la presión[cite: 259].
Reinscripción Supuesta irregularidad en el estatus de inscripción del ciclo escolar[cite: 82]. Forzar una acción rápida para evitar suspensión de materias o acceso a plataforma[cite: 82]. Medio-Alto: Usa una consecuencia seria, aunque puede ser detectado revisando el dominio[cite: 259].

> Los mensajes se diseñaron con tono formal, estructura institucional y consecuencias directas para aumentar la credibilidad inicial[cite: 85, 86].

# 04_EJECUCIÓN_Y_EVIDENCIA_TÉCNICA

Flujo de Configuración en SET (Social Engineer Toolkit) [cite: 88, 92, 93, 94, 95]
# 1. Selección del vector de correo dentro de SET.
set> 5 # Mass Mailer Attack
set:mailer> 1 # E-Mail Attack Single Email Address
set:phishing> 2 # One-Time Use Email Template

# 2. Redacción de asunto, cuerpo HTML y pretexto escolar.
set:phishing> Subject of the email: NOTIFICACIÓN DE SEGURIDAD
set:phishing> Send the message as html or plain? 'h' or 'p' [p]: h
set:phishing> Enter the body of the message, type END (capitals) when finished:
<html>...<body>Estimado usuario es necesario hacer una vinculación de seguridad...</body></html>
set:phishing> END

# 3. Configuración del remitente visible y servidor SMTP.
set:phishing> Send email to: 180370@upslp.edu.mx
set:phishing> 2 # Use your own server or open relay
set:phishing> From address (ex: moo@example.com): 181737@upslp.edu.mx
set:phishing> The FROM NAME the user will see: León Antonio Navarro González
set:phishing> SMTP email server address: 127.0.1.1
set:phishing> Port number for the SMTP server [25]: 1025

# 4. Confirmación de envío.

# 05_ANÁLISIS_DEFENSIVO_E_INDICADORES

> La campaña demuestra que un correo visualmente simple puede ser suficiente para generar riesgo si explota la presión académica[cite: 250]. A continuación se presentan las señales claras que permitirían detectar el ataque[cite: 251]:

Indicador Detectado Observación en la Evidencia Interpretación Defensiva
Remitente no verificable Se simula un nombre institucional, pero la dirección no prueba que el mensaje sea legítimo[cite: 252]. El usuario debe revisar dominio, firma y canal oficial, no confiar solo en el nombre visible[cite: 252].
Cuenta genérica Las variantes usan remitentes con apariencia de área escolar que no corresponden al dominio formal[cite: 252]. Una universidad se comunica desde dominios oficiales y consistentes, no cuentas genéricas[cite: 252].
Lenguaje de urgencia Se menciona bloqueo, límite de tiempo, baja temporal o pérdida de derecho académico[cite: 252]. La urgencia es un disparador diseñado para reducir el análisis crítico del usuario[cite: 252].
Enlace externo/acortado Se observan ligas externas (ej. soporte-verificacion...) y acortadas dentro del mensaje[cite: 252]. Las ligas acortadas y dominios no institucionales deben considerarse altamente sospechosos[cite: 252].
Validación de identidad El correo pide continuar un proceso de vinculación o confirmación dando clic a un enlace[cite: 252]. Cualquier solicitud de inicio de sesión debe verificarse entrando manualmente al portal oficial[cite: 252].

# 06_CONTROLES_RECOMENDADOS_Y_CONCLUSIONES

// CONTROLES PARA USUARIOS

  • > No abrir enlaces de correos que amenacen con consecuencias inmediatas sin verificar en el portal oficial[cite: 262].
  • > Revisar el dominio completo del remitente, no solo el nombre visible[cite: 263].
  • > Desconfiar de enlaces acortados cuando el trámite supuestamente sea institucional[cite: 264].
  • > No ingresar contraseña después de abrir un enlace recibido por correo. Escribir manualmente la URL[cite: 265, 266].
  • > Reportar correos sospechosos al área correspondiente antes de reenviarlos[cite: 267].

// CONTROLES INSTITUCIONALES

  • > Implementar y monitorear SPF, DKIM y DMARC para reducir suplantación de dominio[cite: 269].
  • > Usar campañas periódicas de concientización éticas con ejemplos reales[cite: 270].
  • > Aplicar Autenticación Multifactor (MFA) en portales escolares y cuentas institucionales[cite: 273].
  • > Bloquear o advertir sobre URL acortadas y dominios recién creados[cite: 272].
  • > Mantener plantillas oficiales de comunicación para facilitar el reconocimiento de anomalías[cite: 274].

// CONCLUSIÓN_FINAL_DEL_LABORATORIO

La práctica evidencia que el usuario sigue siendo un punto crítico en la seguridad informática[cite: 289]. Un correo con estructura sencilla puede generar un riesgo inmenso si utiliza elementos de autoridad, urgencia y miedo a consecuencias académicas[cite: 290]. La parte técnica de SET facilita la simulación, pero el éxito del engaño depende principalmente del contexto psicológico del mensaje[cite: 291].

El aprendizaje principal es directo: una institución puede tener excelentes herramientas tecnológicas, pero si sus usuarios carecen de criterios de validación, una campaña de ingeniería social puede comprometer cuentas, información académica o accesos internos[cite: 295]. La defensa no debe depender únicamente de filtros automatizados, sino combinar controles técnicos, capacitación, autenticación multifactor y canales claros de verificación[cite: 293, 294].
